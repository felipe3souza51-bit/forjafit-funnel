# Forja Fit — UI System

## Objetivo
Definir o sistema visual da /oferta para manter consistência, sensação premium e clareza comercial em toda a experiência.

## Direção visual geral
A interface deve parecer:
- escura
- sofisticada
- precisa
- profunda
- premium funcional
- levemente futurista
- limpa
- forte
- sem exagero fitness
- sem cara de template

## Regra central
A interface não existe para “enfeitar”.
Ela existe para reforçar:
- desalinhamento
- clareza
- direção
- progresso possível
- decisão

## Regra de cor
- Verde = direção, alinhamento, ativação do eixo
- Ciano = precisão, sistema, inteligência
- Laranja = ação, CTA, decisão
- Branco/off-white = clareza e leitura
- Cinza frio = contexto, apoio, texto secundário

## Tipografia

### Princípios
- pouca variedade
- alto contraste
- muito controle de peso e espaçamento
- headline com presença
- corpo limpo e respirável

### Hierarquia sugerida
- Display Hero Desktop: 64–76px
- Display Hero Mobile: 38–46px
- H2 Section Desktop: 40–52px
- H2 Section Mobile: 28–34px
- H3 / Cards: 18–24px
- Body Large: 18–20px
- Body Base: 16px
- Microcopy: 13–14px
- Eyebrow: 11–12px, caixa alta, tracking aberto

### Regras tipográficas
- Headlines com line-height curto
- Textos de apoio com line-height confortável
- Evitar blocos longos
- Sempre favorecer leitura rápida + impacto

## Grid e spacing

### Layout base
- container máximo: 1280px
- gutters desktop: 32px
- gutters mobile: 20px
- grid desktop: 12 colunas
- grid tablet: 8 colunas
- grid mobile: 4 colunas

### Espaçamento vertical
- seção hero: 120–160px top/bottom
- seção padrão desktop: 96–128px
- seção padrão mobile: 64–80px
- espaço entre blocos internos: 24 / 32 / 48 / 64

### Regra de ritmo
A página deve respirar.
Poucos elementos por bloco.
Mais peso visual e menos ruído.

## Superfícies e cards

### Linguagem
Não usar “card genérico de SaaS”.
As superfícies devem parecer parte de um sistema de interface premium.

### Padrão visual
- fundo translúcido escuro
- blur leve
- borda suave
- sombra profunda e limpa
- cantos generosos
- camadas internas discretas

### Raios
- raio pequeno: 14px
- raio médio: 20px
- raio grande: 28px
- pill/full: 999px

### Sombras
- shadow-soft: 0 10px 30px rgba(0,0,0,0.22)
- shadow-deep: 0 24px 80px rgba(0,0,0,0.34)
- shadow-focus: 0 0 0 1px rgba(255,255,255,0.05), 0 18px 50px rgba(0,0,0,0.28)

## Eixo de Retomada

## Papel visual
O Eixo de Retomada é o centro da linguagem da oferta.

Ele deve transmitir:
- direção
- alinhamento
- estabilização
- progresso possível

### Elementos do eixo
- linha central dominante
- glow controlado
- núcleo central/silhueta/forma abstrata
- checkpoints orbitais:
  - objetivo real
  - nível atual
  - rotina possível

### Regra visual
No início, a cena pode parecer levemente desalinhada.
Mas nunca bagunçada.
Tem que parecer “desalinhamento inteligente”, não caos.

## Botões

### CTA principal
- cor base: accent-cta
- texto forte
- altura generosa
- raio grande
- sombra limpa
- hover com leve lift
- nunca competir com excesso de brilho ao redor

### CTA secundário
- fundo translúcido
- borda suave
- texto claro
- hover com contraste levemente maior

### Regra
CTA sempre precisa ser facilmente identificado em menos de 1 segundo.

## Ícones e microdetalhes
- usar pouco
- simples
- lineares ou sólidos discretos
- nunca parecer kit genérico fitness
- servir ao entendimento, não decoração

## Blur, brilho e profundidade
- blur deve ser leve e controlado
- glow só para reforçar eixo, foco ou sistema
- evitar neon exagerado
- profundidade deve vir mais de camadas do que de brilho

## Estados interativos

### Hover
- leve elevação
- ajuste sutil de borda
- micro aumento de contraste
- nunca usar salto agressivo

### Active
- compressão mínima
- reforço de foco

### Focus
- contorno elegante e visível
- acessível
- sem quebrar o visual premium

## Motion visual básico
Este arquivo não define a coreografia completa.
Mas toda UI deve aceitar:
- fade
- translate leve
- scale contido
- blur de entrada/saída
- parallax em camadas

## Regras para mobile
- menos glow
- menos profundidade exagerada
- menos camadas simultâneas
- mais contraste
- mais legibilidade
- menos elementos concorrendo

## O que evitar
- gradiente chamativo demais
- neon gamer
- look de dashboard SaaS genérico
- textura demais
- excesso de card
- excesso de texto centralizado
- excesso de borda brilhante
- muito laranja fora dos CTAs
- visual fitness clichê

## Critério de qualidade
Se parecer:
- template premium genérico, está errado
- site gamer, está errado
- landing fitness comum, está errado

Se parecer:
- sistema preciso
- interface cara
- direção clara
- tensão virando controle
então está certo.
